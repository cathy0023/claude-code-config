# rfc-init: RFC 目录初始化器

在项目中创建 RFC 工作流所需的目录结构和索引文件。

## 处理流程

### Step 1: 检测当前项目

从 git 仓库名提取项目标识（如 `coach-react` → `COACH`），作为默认的 Series Prefix。

### Step 2: 创建目录结构

```bash
mkdir -p docs/rfcs/inbox docs/rfcs/draft docs/rfcs/approved docs/rfcs/completed
```

### Step 3: 生成 README.md

```markdown
# RFC Index

This directory tracks Request for Comments documents for this project.

## Series Configuration

| Project | Series Prefix | Next Number |
|---------|--------------|-------------|
| {project_name} | {SERIES_PREFIX} | 1 |

## Directory Structure

| Directory | Purpose |
|-----------|---------|
| `inbox/` | Raw requirement documents (human-placed) |
| `draft/` | RFC drafts (generated by rfc-author) |
| `approved/` | Approved RFCs (moved by mgv-rfc-approve) |
| `completed/` | Completed RFCs (moved after delivery) |

## RFC Status

| ID | Title | Status | File |
|----|-------|--------|------|

## RFC Lifecycle

\`\`\`
inbox → draft → approved → completed
         ↑         ↓
       (rfc-author) (mgv-rfc-approve)
                            ↓
                      rfc-driven-dev
                      (plan → execute → review → fix → archive → deliver)
\`\`\`
```

### Step 4: 追加到 .gitignore（如需要）

检查 `.gitignore` 中是否已排除 RFC 相关条目。RFC 文件应该纳入版本控制，不需要额外排除。

### Step 5: 输出结果

```json
{
  "event": "rfc_initialized",
  "project": "coach-react",
  "series_prefix": "COACH",
  "directories": ["docs/rfcs/inbox", "docs/rfcs/draft", "docs/rfcs/approved", "docs/rfcs/completed"],
  "readme": "docs/rfcs/README.md"
}
```

## Gate

- **通过**：目录结构存在，README.md 已生成
- **阻塞**：docs/rfcs/ 已存在且有内容（避免覆盖）
